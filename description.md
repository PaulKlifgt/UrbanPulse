# UrbanPulse — интеллектуальная платформа оценки городских районов для выбора жилья


## Анализ существующих решений

| Решение          | Оценка района  | Сравнение | Настройка приоритетов | NLP отзывов | Подбор жилья | Охват городов РФ |
| ---------------- | :------------: | :-------: | :-------------------: | :---------: | :----------: | :--------------: |
| ЦИАН             | 1–2 параметра  |    Нет    |          Нет          |     Нет     |      Да      |       Все        |
| ДомКлик          |      Нет       |    Нет    |          Нет          |     Нет     |      Да      |       Все        |
| Яндекс.Карты     |   Визуально    |    Нет    |          Нет          |     Нет     |     Нет      |       Все        |
| 2ГИС             |   Визуально    |    Нет    |          Нет          |     Нет     |     Нет      |      400+        |
| Walk Score (США) |   3 индекса    |    Да     |          Нет          |     Нет     |     Нет      |    0 (не РФ)     |
| **UrbanPulse**   | **8 компонент** |  **Да**  |        **Да**         |   **Да**    |    **Да**    |      **32**      |

Ни одно существующее решение не обеспечивает одновременно: многокомпонентную оценку,
NLP-анализ, настраиваемые веса и интеграцию с рынком недвижимости для российских городов.

## Методы и технологии

### 1. Сбор данных

**Источники:**

- OpenStreetMap (Overpass API) — 17 категорий объектов
- Яндекс.Геокодер API — координаты и границы зон
- Nominatim — fallback-геокодирование

**Обнаружение зон города.** Стратегия «flat zone discovery»: собираются все территориальные
единицы (`admin_level 5–10`, `suburb`, `quarter`, `neighbourhood` из OSM). Дополнительно —
сеточное обратное геокодирование: по регулярной сетке точек (шаг 1.5 км) отправляются
запросы к API, из ответов извлекаются названия зон. Зоны с менее чем 2 попаданиями
отбраковываются. Дедупликация — по порогу расстояния 400 м (формула гаверсинуса).

**Подсчёт инфраструктуры.** Для каждой зоны: количество объектов каждой из 17 категорий
в радиусе 1500 м, нормализация на 1000 жителей.

### 2. Скоринговая модель

**8 компонент:** инфраструктура, образование, здравоохранение, транспорт, экология,
безопасность, досуг, социальная среда.

**Нормализация** — сигмоидная функция:

$$S(x) = \frac{1}{1 + e^{-k \cdot (x - m)}}$$

где *m* — медианное значение, *k* — крутизна.

Обеспечивает:

- нелинейное насыщение (100 магазинов ≠ 10× лучше, чем 10)
- устойчивость к выбросам
- интерпретируемость (0.5 = средний уровень)

**Итоговый индекс:**

$$I = \frac{\sum w_i \cdot S_i}{\sum w_i}$$

где *wᵢ* — настраиваемый вес *i*-й компоненты, *Sᵢ* — нормализованная оценка.

- Диапазон: 0–100
- Грейд: **A** (75+), **B** (60–74), **C** (45–59), **D** (30–44), **F** (<30)

### 3. NLP-модуль

- **Тональность:** словарный метод, 30+ позитивных и 30+ негативных маркеров.
  Score ∈ \[0, 1\].
- **Извлечение проблем:** 10 категорий (мусор, дороги, освещение, шум, парковка,
  озеленение, детская инфраструктура, безопасность, транспорт, ЖКХ) по ключевым словам.
- **Агрегация:** средний sentiment, доля позитива/негатива, топ-3 проблемы на зону.

### 4. ML-кластеризация

`StandardScaler` → `KMeans (k=4)` по 8 компонентам → `PCA (2D)` для визуализации.

Выявляет типологию:

- «центральные насыщенные»
- «спальные»
- «промышленные»
- «пригородные»

### 5. Парсер недвижимости

Selenium + headless Chrome → загрузка карты ЦИАН → перехват fetch-запросов к API кластеров
(JavaScript-инъекция) → извлечение ID объявлений → парсинг страниц (цена, площадь, комнаты,
этаж, адрес, до 5 фото) → валидация → кэширование (6 ч). Thread-safe реализация.

### 6. Стек технологий

`Python 3.10+`, `Streamlit`, `Plotly`, `Pandas`, `NumPy`, `scikit-learn`, `Selenium`,
`geopy`, `Requests`.

## Результаты

### 1. Количественные

| Параметр                    | Значение |
| --------------------------- | :------: |
| Городов                     |    32    |
| Категорий инфраструктуры    |    17    |
| Компонент скоринга          |    8     |
| Зон на крупный город        |  50–120  |
| Зон на средний город        |  20–50   |
| Категорий проблем (NLP)     |    10    |
| Кластеров (ML)              |    4     |

### 2. Функциональные

Действующий прототип веб-приложения:

1. **Карта** — интерактивная (Яндекс.Карты API), цветовая кодировка зон по грейду,
   balloon с детальными оценками и ссылками на жильё.

2. **Рейтинг** — столбчатая диаграмма + ранжированный список.

3. **Анализ зоны** — 8 метрик, радарная диаграмма, проблемы (NLP), предупреждения
   для покупателя с практическими советами, карточки квартир с каруселью фото.

4. **Сравнение** — side-by-side: наложение радаров, таблица с подсветкой лучших значений.

5. **Настройка приоритетов** — слайдеры весов, мгновенный пересчёт индекса.

### 3. Валидация

Сравнение оценок с общеизвестными характеристиками районов: центральные районы
(Вахитовский в Казани, Центральный в СПб) стабильно получают высокие оценки по
инфраструктуре, транспорту и досугу; периферийные — по экологии. Кластерный анализ
выделяет интерпретируемые группы, соответствующие реальной типологии городских территорий.

### 2. Ресурсы

#### Аппаратное обеспечение

- Персональный компьютер для разработки и тестирования.
- **Облачный сервер** — VPS (Virtual Private Server): 2 vCPU, 4 ГБ RAM, 60 ГБ SSD,
  ОС Ubuntu 22.04 LTS. Провайдер — Timeweb Cloud / Selectel (тариф от 600 руб./мес.).
  Используется для развёртывания веб-приложения, хранения кэшированных данных и
  обеспечения круглосуточной доступности сервиса.
- **Доменное имя** — регистрация домена `urbanpulse.ru` (или `.online` / `.app`)
  через регистратор REG.RU / RU-CENTER (от 199 руб./год). SSL-сертификат — Let's Encrypt
  (бесплатный). Обеспечивает профессиональную точку входа и защищённое HTTPS-соединение.

#### API и источники данных

- OpenStreetMap Overpass (бесплатный)
- Яндекс.Геокодер (бесплатный тариф — 1000 запросов/сутки)

#### Программное обеспечение

- Библиотеки Python с открытым исходным кодом: `Streamlit`, `scikit-learn`, `Plotly`,
  `Selenium`, `Pandas`, `NumPy`, `geopy`, `Requests`.
